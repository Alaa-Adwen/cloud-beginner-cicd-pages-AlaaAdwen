What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment. It is a practice that automates the process of building, testing, and deploying code. Every time a developer pushes code, automated checks run to catch bugs early. If everything passes, the code is automatically deployed to a live environment. This removes the need for slow, manual releases and helps teams ship code faster. In cloud systems, CI/CD is essential for keeping deployments reliable and consistent.

My Pipeline

When I push code to the main branch, GitHub Actions automatically runs a workflow. It first checks that all required files exist (index.html, style.css, script.js). If the checks pass, the site files are uploaded as an artifact and then deployed to GitHub Pages — all without any manual steps.
